### 力扣返回二维数组

```c
int **spiralMatrix(int m, int n, struct ListNode *head, int *returnSize, int **returnColumnSizes) {
    // 二级指针模板
    int **res = (int **) malloc(sizeof(int *) * m);
    *returnColumnSizes = (int *) malloc(sizeof(int) * m);
    *returnSize = m;
    for (int i = 0; i < m; ++i) {
        res[i] = (int *) malloc(sizeof(int) * n);
        (*returnColumnSizes)[i] = n;
    }
    // 初始化二维数组
    for (int i = 0; i < m; ++i) {
        for (int j = 0; j < n; ++j) {
            res[i][j] = -1;
        }
    }
    return res;
}
```

### memset

```c
#include <string.h>

int main() {
    int arr[3];
    // memset(dst, val, size)
    // 第三个参数是字节数，以字节为单位向dst开始的地方，填充val
    // 填充0（补码0x00000000）或者-1（补码0xffffffff）都没问题
    // 填充1（补码0x00000001）时，一个int的四个字节变成了0x01010101
    memset(arr, 1, sizeof(arr));
    for (int i = 0; i < 3; ++i) {
        // 16843009 16843009 16843009
        printf("%d ", arr[i]);
    }
    puts("");
    int a = 0x01010101;
    // 16843009
    printf("%d", a);
}
```

- `memset(arr, 0, sizeof(arr));` ：初始化数组为 0 或者 false
- `memset(arr, -1, sizeof(arr));` ：初始化数组为 -1
- `memset(arr, 0x3f, sizeof(arr));` ：初始化数组为无穷大（近似值）

### C 语言常见格式化输入输出

在 C 语言中，格式化输入输出主要依赖标准库 `<stdio.h>` 中的 `printf` 和 `scanf` 函数。通过引入**格式占位符**，编译器能够知道如何将内存中的二进制数据转换为人类可读的文本，或者反过来解析输入。

#### 常见数据类型占位符

| 数据类型            | 占位符       | 说明                                                    | 示例                            |
| :------------------ | :----------- | :------------------------------------------------------ | :------------------------------ |
| **`int`**           | `%d` 或 `%i` | 有符号十进制整数                                        | `printf("%d", 100);`            |
| **`unsigned int`**  | `%u`         | 无符号十进制整数                                        | `printf("%u", 100u);`           |
| **`size_t`**        | `%zu`        | `sizeof` 运算符返回的无符号整型（C99 标准）             | `printf("%zu", sizeof(int));`   |
| **`char`**          | `%c`         | 单个字符                                                | `printf("%c", 'A');`            |
| **`char[]` / 指针** | `%s`         | 字符串（遇到 `\0` 结束）                                | `printf("%s", "Hello");`        |
| **`float`**         | `%f`         | 单精度浮点数（默认保留 6 位小数）                       | `printf("%f", 3.14f);`          |
| **`double`**        | `%lf`        | 双精度浮点数（**printf 可用 %f，但 scanf 必须用 %lf**） | `scanf("%lf", &d);`             |
| **`short`**         | `%hd`        | 短整型                                                  | `printf("%hd", s);`             |
| **`long`**          | `%ld`        | 长整型                                                  | `printf("%ld", l);`             |
| **`long long`**     | `%lld`       | 更长的整型（C99 标准）                                  | `printf("%lld", ll);`           |
| **十六进制**        | `%x` 或 `%X` | 无符号十六进制整数（小写/大写）                         | `printf("%x", 255);` // 输出 ff |
| **八进制**          | `%o`         | 无符号八进制整数                                        | `printf("%o", 8);` // 输出 10   |
| **指针地址**        | `%p`         | 输出变量的十六进制内存地址                              | `printf("%p", &a);`             |

#### printf 进阶：格式修饰符（控制输出样式）

在 `%` 和占位符字母之间，可以插入数字或符号来精细控制输出的宽度、对齐方式和小数位数：

##### 1. 控制浮点数小数位数（`.n`）

* `%.2f`：表示保留 2 位小数（进行四舍五入）。
* `示例`：`printf("%.2f", 3.14159);` $\rightarrow$ 输出 `3.14`

##### 2. 控制输出最小宽度（`n`）

* `%5d`：表示输出至少占 5 个字符位。如果实际数字不足 5 位，**左侧补空格**（默认右对齐）。
* `示例`：`printf("%5d", 12);` $\rightarrow$ 输出 `   12`（前面有 3 个空格）

##### 3. 左对齐修饰符（`-`）

* `%-5d`：表示输出占 5 个字符位，且**左对齐**，不足的位数在右侧补空格。
* `示例`：`printf("%-5d", 12);` $\rightarrow$ 输出 `12   `（后面有 3 个空格）

##### 4. 前导补零（`0`）

* `%05d`：表示输出占 5 个字符位，不足的位数在**左侧补 0**（常用于显示时间、编号）。
* `示例`：`printf("%05d", 12);` $\rightarrow$ 输出 `00012`

#### scanf 输入注意事项

在使用 `scanf` 进行格式化输入时，有几个底层的硬件与逻辑陷阱需要注意：

##### 1. 必须传递变量的地址（`&`）
除了字符串数组名本身就代表地址外，基本类型（如 `int`, `char`, `float`）输入时必须在变量名前加取地址符 `&`。
* `错误`：`scanf("%d", a);`（会导致程序崩溃或内存非法写入）
* ` 正确`：`scanf("%d", &a);`

##### 2. 空格与回车的“过滤”机制
* 当使用 `%d`、`%f`、`%s` 输入时，`scanf` 会**自动跳过**前面的空格、制表符（Tab）和回车。
* **特例 `%c`**：`%c` 会忠实地读取**任何字符**，包括你上一次输入完数字后敲下的“回车键（`\n`）”。
* `解决方案`：在 `%c` 前面加一个空格，写成 `scanf(" %c", &ch);`，这样编译器就会自动帮你过滤掉前面的回车和空格。+


### 生成随机数组

```c
#include <stdio.h>
#include <stdlib.h>
#include <time.h>

int main() {
    int n = 10;
    int arr[10];

    // 1. 初始化随机种子（只调用一次）
    srand(time(NULL));

    // 2. 生成随机数
    for (int i = 0; i < n; i++) {
        arr[i] = rand();  // 0 ~ RAND_MAX
    }

    // 打印
    for (int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }

    return 0;
}
```

#### 封装成函数

```c
#include <stdlib.h>  // rand, srand
#include <time.h>    // time
#include <stdio.h>

void genRandomArray(int* arr, int n, int L, int R) {
    for (int i = 0; i < n; i++) {
        arr[i] = rand() % (R - L + 1) + L;
    }
}

int main() {
    int arr[10];

    srand(time(NULL));

    genRandomArray(arr, 10, -50, 50);

    for (int i = 0; i < 10; i++) {
        printf("%d ", arr[i]);
    }
}
```

### 值返回 vs 指针返回

#### 对比表

| 维度         | 值返回（return struct）         | 指针返回（malloc / new） |
| ------------ | ------------------------------- | ------------------------ |
| 内存位置     | 栈（或寄存器 / 优化后直接构造） | 堆                       |
| 分配开销     | 无                              | 有（malloc / free）      |
| 性能（一般） | 较高（cache 友好）              | 较低（可能 cache miss）  |
| 内存管理     | 自动释放                        | 手动管理，易泄漏         |
| 生命周期     | 局部有效                        | 可跨函数长期存在         |
| 代码复杂度   | 简单                            | 较复杂                   |
| 共享能力     | 不支持共享                      | 支持多个引用             |
| 大对象支持   | 不适合                          | 适合                     |
| 递归场景     | 非常适合                        | 不推荐                   |
| 典型用途     | 状态返回、计算结果              | 数据结构、长期对象       |

值返回并不等价于“有额外拷贝”，在现代编译器下，小对象返回通常是零拷贝（通过 RVO/NRVO），相比之下，malloc/free 的开销更大，且影响 cache。

#### 适合“值返回”

- 对象较小（经验值：< 32~64 bytes）
- 仅用于函数内部计算或返回结果
- 不需要跨函数长期存在
- 不需要共享
- 不涉及资源管理（如动态内存、文件、锁）

典型例子：

```c
Info dfs(...)
Point add(...)
Range merge(...)
```

#### 适合“指针返回”

- 对象较大（如大数组、复杂结构）
- 需要多个地方共享同一对象
- 生命周期跨函数
- 需要原地修改
- 涉及资源管理

典型例子：

```c
TreeNode* newNode(...)
Graph* buildGraph(...)
void update(Obj* o)
```

#### 返回值优化（RVO / NRVO）

返回值优化是编译器的一种优化策略：在调用者的栈空间直接构造返回对象，避免中间拷贝

##### RVO（临时对象）

```c
return (Info){...};
```

##### NRVO（命名返回值）

```c
Info node;
return node;
```

在常见编译器（gcc / clang）+ 优化开启（-O2）时，通常不会发生额外拷贝。即使未优化，也只是一次小对象拷贝。

### 指针数组与数组指针

#### 指针数组

```c
char *arr[3];
```

- `arr` 是一个**长度为 3 的数组**。
- 数组的每个元素是 **`char*` 指针**，可以指向字符串或字符数组。
- 也可称为“指向每行的指针数组”。

假设赋值：

```c
char row0[] = "abc";
char row1[] = "def";
char row2[] = "ghi";

char *arr[3] = {row0, row1, row2};
```

ASCII 图示：

```css
arr[0] --> 'a' 'b' 'c' '\0'
arr[1] --> 'd' 'e' 'f' '\0'
arr[2] --> 'g' 'h' 'i' '\0'
```

- 每行可以独立分配，**不要求内存连续**。
- 访问字符依然可以用 `arr[i][j]`。
- 常用于函数参数是 `char**` 的情况。

```c
char *pa[3] = {"abc", "def", "ghi"};
printf("%c\n", pa[1][2]);  // 输出 'f'
```

#### 数组指针

```c
char (*arr)[3];
```

- `arr` 是一个**指向长度为 3 的字符数组的指针**。
- `*arr` 是一个**数组**，而不是指针。
- 常用于二维数组的函数参数。

假设二维数组：

```c
char grid[3][3] = {{'a','b','c'}, {'d','e','f'}, {'g','h','i'}};
char (*p)[3] = grid;
```

ASCII 图示：

```css
p -> | a b c |
     | d e f |
     | g h i |
```

- 内存连续，**行列固定**。
- 访问字符仍可用 `p[i][j]`。
- `p + 1` 跳过一整行（这里是 3 个字符），而不是一个指针。
- 必须知道每行长度（列数）。

```c
char arr[3][3] = {{'a','b','c'}, {'d','e','f'}, {'g','h','i'}};
char (*p)[3] = arr;
printf("%c\n", p[1][2]); // 输出 'f'
```

#### 区别

| 特性     | 指针数组 `char *arr[3]`  | 数组指针 `char (*arr)[3]`                   |
| -------- | ------------------------ | ------------------------------------------- |
| 本质     | 数组，每个元素是指针     | 指针，指向一个数组                          |
| 内存     | 每行地址独立，不一定连续 | 整块二维数组，连续内存                      |
| 行长度   | 可不一致                 | 必须固定列数                                |
| 访问     | `arr[i][j]`              | `arr[i][j]`                                 |
| 函数参数 | `char **grid`            | `char grid[][cols]` 或 `char (*grid)[cols]` |
