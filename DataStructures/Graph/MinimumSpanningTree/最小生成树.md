## 最小生成树

- 最小生成树（Minimum spanning tree，简称MST）是指在==无向带权图==中选择一些边，在==保证连通性==的情况下，==边的总权值最小==

- 最小生成树不唯一
- 如果无向带权图有 n 个点，最小生成树一定有 n-1 条边


### Kruskal 算法 —— 贪心加“边”

#### 1. 算法流程
1. 把所有的边，根据权值从小到大排序，从权值最小的边开始考虑。
2. 借助**并查集**判断当前边连接的两个顶点是否在同一个集合中：
   * 如果**不在**同一个集合（不会形成环），就把当前边加入 MST，并合并这两个集合。
   * 如果**在**同一个集合（会形成环），则直接跳过该边。
3. 检查是否已经加入了 n-1 条边（或者考察完所有的边），即可得到 MST。

#### 2. 核心思想
**“管它连不连通，只挑全局最短的边。”** 每次寻找不会导致出现环的权值最小的边，依次加入到 MST 中。图的形态由“孤立的点 $\rightarrow$ 许多独立的小树（森林） $\rightarrow$ 最终合并为一棵大树”。

#### 3. 复杂度分析
*   **时间复杂度**：$$\mathcal{O}(M \log M)$$
    *   **输入与初始化**：$\mathcal{O}(N + M)$
    *   **边权排序（大头）**：$\mathcal{O}(M \log M)$
    *   **并查集遍历**：$\mathcal{O}(M \cdot \alpha(N))$，由于路径压缩优化，$\alpha(N)$ 近似常数 $\mathcal{O}(1)$，故遍历耗时 $\mathcal{O}(M)$。
    *   *总复杂度由排序决定，即 $\mathcal{O}(M \log M)$。在稀疏图中表现极佳。*
*   **空间复杂度**：$$\mathcal{O}(N + M)$$（取决于存储边和并查集的数组大小）

### Prim 算法 —— 贪心加“点”

#### 1. 算法流程（堆优化版）
1. 声明两个集合：解锁的点的集合叫 `set`（通常用 `visit` 数组实现），解锁的边的集合叫 `heap`（小顶堆/优先队列），初始状态均为空。
2. 从任意点开始：将开始点加入 `set`，把该点的所有邻边加入到 `heap` 中。
3. 从 `heap` 中弹出权值最小的边 e，查看边 e 去往的终点 x：
   * 如果 x **已经在** `set` 中，说明会形成环，舍弃边 e，重复步骤 3。
   * 如果 x **不在** `set` 中，说明找到了新顶点，边 e 属于 MST。将 x 加入 `set`，并将 x 的所有**新邻边**加入 `heap`，重复步骤 3。
4. 当 `heap` 为空时，MST 构建完成。

#### 2. 核心思想
**将图分为两个切面**：组成 MST 的点构成的集合 A，和剩下元素构成的集合 B。每次从集合 B 中找到**距离集合 A 最近（边权最小）**的点，拉入集合 A 中。图的形态由“一个点 $\rightarrow$ 一棵逐渐变大的树 $\rightarrow$ 最终覆盖全图”。

#### 3. 复杂度分析
*   **时间复杂度**：$$\mathcal{O}(M \log M)$$ （或 $\mathcal{O}(M \log N)$）
    *   **建图**：$\mathcal{O}(N + M)$
    *   **堆操作**：每条边最多进堆、出堆一次，小顶堆调整的复杂度为 $\log M$。整个过程中堆操作的总复杂度为 $\mathcal{O}(M \log M)$。
    *   *总复杂度为 $\mathcal{O}(M \log M)$。如果在稠密图（$M$ 接近 $N^2$）中，可采用索引堆优化至 $\mathcal{O}(M \log N)$，甚至使用斐波那契堆优化至 $\mathcal{O}(M + N \log N)$。*
*   **空间复杂度**：$$\mathcal{O}(N + M)$$（取决于邻接表和堆的大小）

### 索引堆优化的 Prim 算法

#### 传统 Prim 的致命痛点：堆被“灌水”
传统的 Prim 算法，核心思路是**“把所有候选的【边】扔进堆里”**。每当一个新节点加入最小生成树（MST），我们就会把它带出来的所有邻接边一股脑地推入堆中。这就导致了一个严重后果：堆的体积取决于边的数量（最大可达 M）。
如果一个图有 5000 个点，却有 400,000 条边：
• 堆里面会塞满成百上千条去往同一个目的地的“冗余边”。
• 每次从堆里弹出一个元素或放入一个元素，堆维护的时间复杂度是 O(log M)。
• 当边数 M 极大时，堆不仅臃肿、耗费巨大内存，还会因为频繁的元素交换导致大量的运行时间被浪费。
总结来说：传统 Prim 是“边入堆”，在边极其稠密的图里，堆太大，跑得慢。

#### 破局思路：能不能让“点入堆”？
为了解决堆太臃肿的问题，我们产生了一个极其自然的直觉：既然最小生成树最终的目标是连接所有【节点】，那为什么不让堆里只存放【节点】，让堆的最大体积被死死限制在节点数 N 以内呢？

如果让点入堆，游戏规则就变了：
• 堆里不再存放具体的边，而是存放**“目前尚未加入 MST 的节点”**。
• 堆排序的依据，是这些节点**“当前已知距离 MST 的最短距离（dist）”**。
• 这样一来，不管图里有几百万条边，堆里的元素个数永远不可能超过节点数 N。堆的单次维护成本直接从 O(log M) 骤降到 O(log N)。

#### 引出“反向索引表”的必然性
然而，当真正去实现“点入堆”时，你会立刻撞上一个传统二叉堆无法解决的工程难题：假设节点 v 已经在堆里了，当前记录它离 MST 的距离是 10。此时，MST 新加入了一个节点 u，发现 u 有一条连向 v 的边，权重只有 3。
按照贪心策略，我们应该把 v 在堆里的距离从 10 动态修改为 3，然后对它进行一次**上滤（percolate_up）**来调整它在堆里的位置。
矛盾来了： 二叉堆是一个一维数组。你只知道 v 这个节点的名字，但根本不知道它此时此刻在堆数组的哪个下标上！
如果为了找 v 而去把整个堆数组遍历一遍，复杂度就会退化成 O(N)，那用堆优化的意义就彻底荡然无存了。
为了解决**“如何用 O(1) 时间在堆里瞬间定位某个节点”的问题，必须引入一个外挂数据结构——反向索引表（pos 数组）**。

#### 这个优化究竟是为了什么？
引入“带反向索引表的索引堆优化”，核心目的只有一个：在确保堆体积只有 O(N) 的前提下，实现对堆内节点 O(1) 时间的精准定位与原地修改。

通过引入 pos 数组（pos[v] 记录节点 v 当前在堆数组中的下标），整个算法的思路得以完美连贯：

1. 图很大，边很多 -> 放弃传统的“边入堆”，改为“点入堆”，将堆的最大体积锁定在 N。
2. 点入堆后需要动态更新距离 -> 利用反向索引表 pos，在查找到更短的边时，瞬间定位到该点在堆中的位置。
3. 原地调整 -> 找到下标后，直接改写它的距离值，并当场触发上滤，花 O(log N) 的代价完成堆的翻新。
   这就是为什么要引入这个优化。它不是为了好玩或者炫技，而是为了通过空间换时间，彻底斩断边数 M 对堆体积的绑架，从而让 Prim 算法在面对“节点少、边极其多”的稠密图时，能够具备降维打击般的运行效率。

### 算法选型

| 算法名称    | 核心驱动             | 适用场景               | 关键数据结构                   | 防环机制                        |
| :---------- | :------------------- | :--------------------- | :----------------------------- | :------------------------------ |
| **Kruskal** | 边驱动（排好序硬怼） | **稀疏图**（边少点多） | 边集数组 + **并查集**          | 并查集判断 `find(u) == find(v)` |
| **Prim**    | 点驱动（向外圈地）   | **稠密图**（边多点少） | 邻接表 + **小顶堆** + 布尔数组 | 布尔数组/Set 判断 `visited[x]`  |

为什么 Prim 用 Set，Kruskal 用并查集？

*   **Prim 算法是“单点抱团”**：它从一个点出发，始终维持**一棵唯一的树**。所有已解锁的点都在同一个集合（`Set A`）中。判断一条边是否可用，只需要看它的终点“是否在这个唯一的集合中”（即 `visited` 标记）。
*   **Kruskal 算法是“多点开花”**：它每次只挑全局最短的边，在图的各处同时生成多棵独立的小树（**森林**）。因为存在多棵树，光靠一个 `Set` 或 `visit` 数组无法分辨两个点是“属于同一棵树”还是“属于两棵不同的树”，所以**必须借助并查集**来合并和判断集合归属。

### [P3366 【模板】最小生成树](https://www.luogu.com.cn/problem/P3366)

- Kruskal 算法

```c++
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>

using namespace std;

// 并查集中存放的是点的编号
vector<int> father;

void build(int n) {
    father.resize(n + 1);
    // 顶点下标从 1 开始
    for (int i = 1; i <= n; ++i)
        father[i] = i;
}

int find(int i) {
    if (i != father[i])
        father[i] = find(father[i]);
    return father[i];
}

bool isSameSet(int a, int b) {
    return find(a) == find(b);
}

void un1on(int a, int b) {
    int fa = find(a);
    int fb = find(b);
    if (fa == fb) return;
    father[fa] = fb;
}

int main() {
    int n, m;
    cin >> n >> m;
    vector<vector<int>> edges(m);
    for (int i = 0; i < m; ++i) {
        edges[i].resize(3);
        cin >> edges[i][0] >> edges[i][1] >> edges[i][2];
    }

    // 按照边的权重排序
    sort(begin(edges), end(edges),
         [](vector<int> &a, vector<int> &b) { return a[2] < b[2]; });

    // 构建并查集
    build(n);
    // mst 的权重
    int weight = 0;
    // 已经加入 mst 的边数
    int count = 0;
    // 凑够 n - 1 条边就可以退出
    for (int i = 0; i < m && count < n - 1; ++i) {
        // 两个点在一个集合里，说明已经他俩之间已经有路径了，如果再把当前边加入 mst 就会出现环，所以跳过
        if (isSameSet(edges[i][0], edges[i][1])) continue;
        // 把两个顶点加入 mst
        un1on(edges[i][0], edges[i][1]);
        // 累加 mst 权重
        weight += edges[i][2];
        count++;
    }

    if (count == n - 1) {
        cout << weight;
    } else {
        cout << "orz";
    }
}
```

- Prim 算法

```c++
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>

using namespace std;

// 标记顶点是否已经加入 MST 的集合中
vector<bool> visited;
// 小顶堆，下标从 0 开始，pair<点的下标, 边的权值>
vector<pair<int, int>> heap;
int lenOfHeap;

// 顶点下标从 1 开始
void build(int n, int m) {
    visited.resize(n + 1, false);
    // 堆的最大容量就是边数的两倍，tmd，找了半天错才发现不是 m
    heap.resize(m * 2);
    // 堆的初始容量为 0
    lenOfHeap = 0;
}

// curIndex 为要调整位置的元素的下标
void adjustHeap(int curIndex) {
    auto temp = heap[curIndex];
    int leftChild = 2 * curIndex + 1;
    while (leftChild <= (lenOfHeap - 1)) {
        if (leftChild < (lenOfHeap - 1)
            && heap[leftChild].second > heap[leftChild + 1].second)
            leftChild++;
        if (heap[leftChild].second >= temp.second) break;
        heap[curIndex] = heap[leftChild];
        curIndex = leftChild;
        leftChild = curIndex * 2 + 1;
    }
    heap[curIndex] = temp;
}

void pushToHeap(pair<int, int> p) {
    heap[lenOfHeap] = p;
    int curIndex = lenOfHeap;
    int parentIndex = (curIndex - 1) / 2;
    lenOfHeap++;
    while (parentIndex >= 0) {
        if (heap[parentIndex].second <= p.second) break;
        heap[curIndex] = heap[parentIndex];
        curIndex = parentIndex;
        if (curIndex == 0) break;
        parentIndex = (curIndex - 1) / 2;
    }
    heap[curIndex] = p;
}

pair<int, int> getHeapTop() {
    auto res = heap[0];
    heap[0] = heap[lenOfHeap - 1];
    lenOfHeap--;
    adjustHeap(0);
    return res;
}

int main() {
    int n, m;
    cin >> n >> m;
    vector<vector<int>> edges(m);
    for (int i = 0; i < m; ++i) {
        edges[i].resize(3);
        cin >> edges[i][0] >> edges[i][1] >> edges[i][2];
    }

    // 建堆和 visited 数组
    build(n, m);

    // 邻接表存放带权值的无向图
    vector<vector<pair<int, int>>> graph(n + 1);
    for (const auto &item: edges) {
        graph[item[0]].emplace_back(make_pair(item[1], item[2]));
        graph[item[1]].emplace_back(make_pair(item[0], item[2]));
    }

    // 任意一点作为初始点加入集合
    visited[1] = true;
    // 把初始点的相关边都入堆
    for (const auto &item: graph[1])
        // 边的另一个点的下标和边的权值入堆
        pushToHeap(make_pair(item.first, item.second));

    // MST 的权值
    int weight = 0;
    // 已经加入 MST 的边数
    int count = 0;
    // 每次选出一个与 MST 集合距离最短的顶点加入集合
    while (lenOfHeap != 0) {
        // 堆顶就是到 MST 集合距离最短的
        auto top = getHeapTop();
        // 已经在 MST 集合中，就跳过
        if (visited[top.first] == true) continue;
        // 否则把这个点加入集合
        visited[top.first] = true;
        // 累加 MST 权值
        weight += top.second;
        count++;
        // 再把与 top.first 相关的边入堆
        for (const auto &item: graph[top.first])
            pushToHeap(make_pair(item.first, item.second));
    }

    if (count == n - 1)
        cout << weight;
    else
        cout << "orz";
}
```

```c++
#include <iostream>
#include <vector>
#include <queue>
#include <unordered_set>
#include <algorithm>

using namespace std;

struct cmp {
    bool operator()(pair<int, int> &p1, pair<int, int> &p2) {
        return p1.second > p2.second;
    }
};


int main() {
    int n, m;
    cin >> n >> m;
    vector<vector<int>> edges(m);
    for (int i = 0; i < m; ++i) {
        edges[i].resize(3);
        cin >> edges[i][0] >> edges[i][1] >> edges[i][2];
    }

    // pair<点的下标, 边的权值>
    priority_queue<pair<int, int>, vector<pair<int, int>>, cmp> heap;
    // 标记顶点是否已经加入 MST 的集合中
    unordered_set<int> st;

    // 邻接表存放带权值的无向图
    vector<vector<pair<int, int>>> graph(n + 1);
    for (const auto &item: edges) {
        graph[item[0]].emplace_back(make_pair(item[1], item[2]));
        graph[item[1]].emplace_back(make_pair(item[0], item[2]));
    }

    // 任意一点作为初始点加入集合
    st.emplace(1);
    // 把初始点的相关边都入堆
    for (const auto &item: graph[1])
        // 边的另一个点的下标和边的权值入堆
        heap.push(make_pair(item.first, item.second));

    // MST 的权值
    int weight = 0;
    // 已经加入 MST 的边数
    int count = 0;
    // 每次选出一个与 MST 集合距离最短的顶点加入集合
    while (!heap.empty()) {
        // 堆顶就是到 MST 集合距离最短的
        auto top = heap.top();
        heap.pop();
        // 已经在 MST 集合中，就跳过
        if (st.find(top.first) != st.end()) continue;
        // 否则把这个点加入集合
        st.emplace(top.first);
        // 累加 MST 权值
        weight += top.second;
        count++;
        // 再把与 top.first 相关的边入堆
        for (const auto &item: graph[top.first])
            heap.push(make_pair(item.first, item.second));
    }

    if (count == n - 1)
        cout << weight;
    else
        cout << "orz";
}
```

- Prim 算法优化


```c++
#include <stdbool.h>
#include <stdio.h>
#include <string.h>

#define MAXN 5005
#define MAXM 400005

// 链式前向星存图
int head[MAXN];
int to[MAXM];
int next[MAXM];
int weight[MAXM];
int edge_cnt;

// heap[i] = u：堆中第 i 个位置存放的是顶点编号 u (heap 下标从 0 开始)
int heap[MAXN];
int heap_sz;

// dist[u] = d：顶点 u 到当前 MST 集合的最短距离 (dist 下标从 1 开始)
// 整个小顶堆的大小比较严格基于 dist[heap[i]] 的值
int dist[MAXN];

// pos[u] = i：反向索引表。顶点 u 当前在 heap 数组中的下标是 i
// 状态约定：
// -1 ：尚未进入堆中（初始状态）
// -2 ：已经加入了 MST 集合（已出堆）
// >=0：顶点在堆中的具体下标
int pos[MAXN];

void init_heap() {
    heap_sz = 0;
    memset(pos, -1, sizeof(pos));
}

void percolate_up(int top, int idx) {
    int v = heap[idx];
    int d = dist[v];
    int cur = idx;
    int parent = (cur - 1) >> 1;

    while (cur > top && dist[heap[parent]] > d) {
        heap[cur] = heap[parent];
        // 同步更新反向索引表
        pos[heap[cur]] = cur;
        cur = parent;
        parent = (cur - 1) >> 1;
    }

    heap[cur] = v;
    pos[v] = cur;
}

void heapify(int idx) {
    int v = heap[idx];
    int cur = idx;
    int lc = (cur << 1) + 1;

    while (lc < heap_sz) {
        if (lc + 1 < heap_sz && dist[heap[lc + 1]] < dist[heap[lc]]) ++lc;
        heap[cur] = heap[lc];
        pos[heap[cur]] = cur;
        cur = lc;
        lc = (cur << 1) + 1;
    }
    heap[cur] = v;
    pos[v] = cur;
    percolate_up(idx, cur);
}

void push(int v) {
    int heap_idx = heap_sz++;
    heap[heap_idx] = v;
    pos[v] = heap_idx;
    percolate_up(0, heap_idx);
}

int pop() {
    int u = heap[0];
    // 只要出堆，立刻打上 -2 标签，代表正式委任进 MST 集合
    pos[u] = -2;
    --heap_sz;
    if (heap_sz > 0) {
        heap[0] = heap[heap_sz];
        pos[heap[0]] = 0;
        heapify(0);
    }
    return u;
}

void init_graph(int n) {
    edge_cnt = 1;
    memset(head, 0, sizeof(int) * (n + 1));
}

void add_edge(int u, int v, int w) {
    int edge_idx = edge_cnt++;
    next[edge_idx] = head[u];
    to[edge_idx] = v;
    weight[edge_idx] = w;
    head[u] = edge_idx;
}

int main() {
    int n, m;
    if (scanf("%d%d", &n, &m) != 2) return 0;

    init_graph(n);
    init_heap();

    for (int i = 0; i < m; ++i) {
        int u, v, w;
        scanf("%d%d%d", &u, &v, &w);
        add_edge(u, v, w);
        add_edge(v, u, w);
    }

    int res = 0;
    int cnt = 0;
    // 任意挑选 1 号节点作为树的根节点，将其距离设为 0 并推入堆
    dist[1] = 0;
    push(1);

    while (heap_sz > 0 && cnt < n) {
        // 弹出当前距离 MST 集合最近的顶点 v
        int v = pop();
        res += dist[v];
        ++cnt;

        // 考察从 v 出发的每一条邻接边
        for (int e = head[v]; e > 0; e = next[e]) {
            int vv = to[e];
            if (pos[vv] == -2) {
                // P1：vv 已经加入 MST 集合
                continue;
            } else if (pos[vv] == -1) {
                // P2：vv 首次被发现，从未进过堆
                dist[vv] = weight[e];
                push(vv);
            } else if (pos[vv] >= 0) {
                // P3：vv 目前已经在堆里
                // 如果这次发现的新边权重，比它之前记录的旧距离还要短
                if (weight[e] < dist[vv]) {
                    dist[vv] = weight[e];  // 改写更优的距离值
                    // 索引堆：利用 pos[vv] 以 O(1) 复杂度得到该点在堆中的下标
                    percolate_up(0, pos[vv]);
                }
            }
        }
    }

    // 判定是否能生成覆盖全图的树
    if (cnt == n) {
        printf("%d\n", res);
    } else {
        printf("orz\n");
    }

    return 0;
}
```

### [1697. 检查边长度限制的路径是否存在](https://leetcode.cn/problems/checking-existence-of-edge-length-limited-paths/)

```c++
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>

using namespace std;

class Solution {
public:
    vector<int> father;

    void build(int n) {
        father.resize(n);
        for (int i = 0; i < n; ++i)
            father[i] = i;
    }

    int find(int i) {
        if (i != father[i])
            father[i] = find(father[i]);
        return father[i];
    }

    bool isSameSet(int a, int b) {
        return find(a) == find(b);
    }

    void un1on(int a, int b) {
        int fa = find(a);
        int fb = find(b);
        if (fa == fb) return;
        father[fa] = fb;
    }

    // 顶点下标从 0 开始
    vector<bool> distanceLimitedPathsExist(int n, vector<vector<int>> &edgeList, vector<vector<int>> &queries) {
        // 将边按照权值排序
        sort(begin(edgeList), end(edgeList),
             [](vector<int> &a, vector<int> &b) { return a[2] < b[2]; });

        int len = queries.size();
        vector<vector<int>> questions(len, vector<int>(4));
        for (int i = 0; i < len; ++i) {
            questions[i][0] = queries[i][0];
            questions[i][1] = queries[i][1];
            questions[i][2] = queries[i][2];
            // 多加一条记录初始问题的位置
            questions[i][3] = i;
        }
        // 也按照边长度限制进行排序
        sort(begin(questions), end(questions),
             [](vector<int> &a, vector<int> &b) { return a[2] < b[2]; });

        // 构建并查集
        build(n);
        vector<bool> res(len);
        for (int i = 0, edgeIndex = 0; i < len; ++i) {
            // 问题已按照边长度限制的大小排序，每次把小于这次边限制的所有边生成 MST，查看要求的两个点是否都在 MST 中
            // 也就是每次把点合并到 MST 集合上，然后判断要求的两个点是否属于同一个结合
            while (edgeIndex < edgeList.size() && edgeList[edgeIndex][2] < questions[i][2]) {
                un1on(edgeList[edgeIndex][0], edgeList[edgeIndex][1]);
                edgeIndex++;
            }
            res[questions[i][3]] = isSameSet(questions[i][0], questions[i][1]);
        }

        return res;
    }
};
```

### [P2330 [SCOI2005] 繁忙的都市](https://www.luogu.com.cn/problem/P2330)

- 最小瓶颈树：使图连通，且最大边尽量小
- 最小生成树一定是最小瓶颈树

```c++
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>

using namespace std;

vector<int> father;

void build(int n) {
    father.resize(n + 1);
    for (int i = 1; i <= n; ++i)
        father[i] = i;
}

int find(int i) {
    if (i != father[i])
        father[i] = find(father[i]);
    return father[i];
}

bool isSameSet(int a, int b) {
    return find(a) == find(b);
}

void un1on(int a, int b) {
    int fa = find(a);
    int fb = find(b);
    if (fa == fb) return;
    father[fa] = fb;
}

int main() {
    int n, m;
    cin >> n >> m;
    vector<vector<int>> edges(m, vector<int>(3));
    for (int i = 0; i < m; ++i)
        cin >> edges[i][0] >> edges[i][1] >> edges[i][2];

    // 按照边的权值排序
    sort(begin(edges), end(edges),
         [](vector<int> &a, vector<int> &b) { return a[2] < b[2]; });

    build(n);

    int count = 0;
    int res;
    for (int i = 0; i < m && (count < n - 1); ++i) {
        if (isSameSet(edges[i][0], edges[i][1])) continue;
        un1on(edges[i][0], edges[i][1]);
        count++;
        if (count == n - 1) res = edges[i][2];
    }

    cout << n - 1 << " " << res;
}
```
